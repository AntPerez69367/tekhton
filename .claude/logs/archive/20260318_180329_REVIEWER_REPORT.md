- `_match_pattern()` uses `echo "$text"` (errors.sh:37), which can mishandle inputs beginning with `-n` or containing `\t`/`\n` escape sequences on some `echo` implementations. Consider `printf '%s\n' "$text"` for consistency with the now-fixed `redact_sensitive()`.
