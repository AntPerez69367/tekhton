## Suspected Root Cause Areas

1. **test_agent_summary.sh header mismatch** — Lines 6-10 declare 5 test scenarios in comment, but actual test blocks (lines 47-314) implement different groupings. The comment lists "Success run", "Failure with error classification (category)", "Failure with error classification (recovery)", "Redaction", "Unicode/ASCII" — but test blocks show success, failure, null-run, tail-friendly, redaction-integration, and Unicode/ASCII fallback. Needs comment alignment with actual test structure.

2. **report_error() border rendering** — Lines 77, 80-81, 83-84 use bare `echo "${_box_v} content"` with no right border, while line 78-79 uses `printf` with right border `$_box_v`. Creates visually incomplete box (open right side). All non-border lines should use `printf` pattern for consistency.

3. **stderr file capture divergence** — Spec (CLAUDE.md M12.2) says redirect agent stderr to dedicated `$SESSION_TMP/agent_stderr.txt`, but implementation (lib/agent_monitor.sh:95) uses `> "$_fifo" 2>&1` merging stderr into FIFO. Ring buffer captures everything, but dedicated stderr file is never populated from real stderr — only synthetic API error hint is written when `_API_ERROR_DETECTED=true`. Works in practice but diverges from spec intent for non-API errors.
