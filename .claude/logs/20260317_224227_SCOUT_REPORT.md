## Suspected Root Cause Areas

### Item 1: Missing clarifying comment
- **Location**: `lib/milestone_archival.sh:187` (inside awk block)
- **Issue**: The asymmetry between `heading_level = RLENGTH` (matches `^#{1,5}`) and `this_level = RLENGTH - 1` (matches `^#{1,5}[[:space:]]`) is correct but confusing without explanation
- **Cause**: The second regex includes trailing whitespace, so RLENGTH is one character larger; subtracting 1 makes the comparison valid
- **Impact**: Low risk (code works correctly), but future maintainers may waste time debugging or "fixing" this intentional asymmetry

### Item 2: Fragile mktemp fallback
- **Location**: `lib/milestone_archival.sh:174–176`
- **Issue**: `mktemp "${tmp_dir}/archival_XXXXXX"` will fail silently if `${tmp_dir}` is read-only (e.g., project in read-only directory or misconfigured TEKHTON_SESSION_DIR)
- **Current fallback**: Lines 175 uses `${TEKHTON_SESSION_DIR:-$(dirname "$claude_md")}` which tries dirname of CLAUDE.md as fallback, but this doesn't guarantee a writable directory
- **Cause**: No secondary fallback to system temp directory (`/tmp`, `/var/tmp`) if the primary temp location fails
- **Impact**: Archival silently fails, leaving milestone definitions in CLAUDE.md instead of moving to archive; repeat attempts re-archive same milestone (idempotency prevents corruption but archival doesn't complete)
