Implement Milestone 15.1: Bug Fixes — Notes Gating, Resolved Cleanup, and Auto-Commit Default

Three independent systemic bug fixes:
1. **Notes Gating** — gate `claim_human_notes()` behind task matching
2. **Resolved Cleanup** — clear `## Resolved` section in NON_BLOCKING_LOG.md

## Suspected Root Cause Areas

- **Phantom HUMAN_NOTES injection**: `claim_human_notes()` is always called when HUMAN_NOTE_COUNT > 0 (stages/coder.sh line 341-343), regardless of whether the task is actually about addressing human notes. Fix: add `should_claim_notes()` gating.
- **COMPLETE→IN PROGRESS downgrade phantom**: stages/coder.sh lines 440-452 force rework when `HUMAN_NOTES_ALL_ADDRESSED=false`, but this variable is not being set/tracked correctly when notes aren't claimed. Fix: remove this block entirely since notes gating prevents the phantom injection.
- **NON_BLOCKING_LOG ## Resolved growth**: lib/drift_cleanup.sh has `clear_completed_nonblocking_notes()` for the ## Open section but no equivalent for ## Resolved section. Resolved items accumulate indefinitely. Fix: add `clear_resolved_nonblocking_notes()` following the same pattern.
- **AUTO_COMMIT defaulting to false in milestone mode**: lib/config_defaults.sh line 128 hardcodes `AUTO_COMMIT=false`. In milestone mode, auto-advance and autonomous loops need auto-commit by default. Fix: conditional assignment after MILESTONE_MODE is determined.
- **--with-notes flag missing**: tekhton.sh lacks `--with-notes` flag parsing. Users cannot force notes to be claimed on tasks that don't mention HUMAN_NOTES. Fix: add flag parsing around line 460 to set `WITH_NOTES=true`.
