## Suspected Root Cause Areas

The implementation appears nearly complete. Potential areas for validation:

- **Ring buffer edge cases**: Verify correct handling of wraparound when buffer fills beyond 50 lines; confirm no memory leaks on long-running agents
- **Stderr redirection flow**: The spec requires stderr redirection to `$SESSION_TMP/agent_stderr.txt`, but agent_monitor.sh redirects stderr to FIFO via `2>&1` (line 95). Verify this captures all stderr properly for classify_error
- **API error pattern matching**: Real-time detection uses case/grep patterns (lines 142-159) — ensure all Anthropic API error formats are caught (rate_limit_error, overloaded_error, server_error, authentication_error)
- **UPSTREAM bypass in null-run path**: Lines 196-210 of agent.sh skip null-run classification for UPSTREAM errors, but verify this doesn't mask actual null runs in mixed scenarios
- **Subshell variable export**: API error detection occurs in subshell (line 253-256 of agent_monitor.sh), flags written to file, parent reads them — verify this IPC pattern is bulletproof across platforms
- **Acceptance criteria validation**: No test files found specifically testing M12.2 acceptance criteria (API error detection, ring buffer, error classification flow)
