## Review Notes

The fix is correct and minimal. `declare -x "name=value"` inside a bash function creates a
**function-local** variable (the `-x` flag only adds the export attribute to the local copy —
it does not propagate the assignment to the calling scope). Replacing with
`export "name=value"` assigns to the global shell environment, which is visible to all
callers. Both occurrences (line 49 in `_filter_block`, line 132 in `_compress_if_over_budget`)
are fixed identically.
