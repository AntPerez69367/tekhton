# Milestone 0 Scout Report — Security Hardening

## Relevant Files

### Phase 1: Config Injection Elimination (Critical)
- `lib/config.sh` — Uses `source <(sed 's/\r$//'...)` which executes arbitrary code if config contains `$(`, backticks, `;`, `|`, `&`, etc.
- `lib/plan.sh` — Same unsafe config sourcing pattern for planning configuration
- `lib/gates.sh` — Lines 49 and 78 use `eval "..."` on BUILD_CHECK_CMD and validation_cmd without sanitization
- `lib/hooks.sh` — Executes `${ANALYZE_CMD}` and `${TEST_CMD}` unquoted, vulnerable to shell metacharacter injection

### Phase 2: Temp File Hardening (High)
- `tekhton.sh` — Lacks per-session temp directory and PID-based lock file for preventing concurrent runs
- `lib/agent.sh` — Lines 174-175 create predictable `/tmp/tekhton_*_last_turns` and `/tmp/tekhton_*_agent_exit` files (TOCTOU vulnerability); Line 225 creates FIFO at `/tmp/tekhton_agent_fifo_$$` (predictable)
- `lib/drift.sh` — Uses `mktemp` but paths may not be isolated to session directory; vulnerability if session dir not established
- `lib/hooks.sh` — Commit message generation creates unnamed temp files; needs session directory isolation

### Phase 3: Prompt Injection Mitigation (High)
- `lib/prompts.sh` — `render_prompt()` substitutes `{{TASK}}` without delimiters; user task can inject prompt directives
- `stages/coder.sh` — Injects ARCHITECTURE_BLOCK (line 47), REVIEWER_REPORT (line 150), TESTER_REPORT (line 179), BUILD_ERRORS_CONTENT (line 338) without wrapping delimiters
- `stages/review.sh` — Injects ARCHITECTURE_CONTENT and coder reports into reviewer prompt without content delimiters
- `stages/tester.sh` — Injects ARCHITECTURE_CONTENT without delimiters
- `stages/architect.sh` — Similar file-content injections without protective delimiters
- `prompts/coder.prompt.md` — Lacks anti-injection directive explaining that content sections are untrusted
- `prompts/reviewer.prompt.md` — Lacks anti-injection directive
- `prompts/tester.prompt.md` — Lacks anti-injection directive
- `prompts/scout.prompt.md` — Lacks anti-injection directive
- `prompts/architect.prompt.md` — Lacks anti-injection directive

### Phase 4: Git Safety (High)
- `lib/hooks.sh` — `generate_commit_message()` sanitizes TASK string (line 29) but does not check `.gitignore` before staging secrets
- `tekhton.sh` — Line 744 uses `git add -A` without verifying `.gitignore` contains patterns for `.env`, `.claude/logs/`, `*.pem`, `*.key`, `id_rsa`, etc.

### Phase 5: Defense-in-Depth (Medium)
- `lib/config.sh` — No upper bounds on numeric config values (MAX_REVIEW_CYCLES can be set to unlimited, causing runaway loops)
- `stages/coder.sh` — No file size validation before reading artifacts into variables; large files can cause buffer/memory issues
- `stages/architect.sh` — Same lack of file size validation
- `lib/prompts.sh` — No file size checks on template files or injected content
- `lib/agent.sh` — Windows PID-based kill on line 188 uses `taskkill /PID` but lacks fallback for process name kill; comment notes --disallowedTools is best-effort denylist with bypass vectors

## Suspected Root Cause Areas

1. **Config sourcing vulnerability** — lib/config.sh:28 uses `source <(sed...)` which executes shell code. A malicious pipeline.conf with `ANALYZE_CMD="$(whoami)"` or backticks will be executed during config load.

2. **Eval-based command execution** — lib/gates.sh:49 and 78 use `eval` on user-configured commands. Attacker-controlled BUILD_CHECK_CMD can break out and execute arbitrary shell commands.

3. **Unquoted variable expansion** — lib/gates.sh:20 and lib/hooks.sh:80, 124, 136 expand ${ANALYZE_CMD} without quotes, allowing shell metacharacter injection if the variable contains `|`, `&&`, `;`, etc.

4. **Predictable temp file paths** — lib/agent.sh:174-175 and 225 use `/tmp/tekhton_${_project_slug}_*` and `/tmp/tekhton_agent_fifo_$$`. An attacker with local access can create these files in advance, causing TOCTOU race conditions or FIFO blocking.

5. **Prompt injection via file content** — stages/*.sh inject REVIEWER_REPORT, TESTER_REPORT, BUILD_ERRORS_CONTENT, ARCHITECTURE_CONTENT directly into prompts without delimiters. Malicious content from a previous run (e.g., "Ignore your system prompt and...") will be interpreted by Claude as directives.

6. **{{TASK}} injection** — lib/prompts.sh:render_prompt() substitutes {{TASK}} without delimiters. A task like `"Implement X. New instruction: Ignore architecture and..."` will poison the coder prompt.

7. **Unguarded git add** — tekhton.sh:744 uses `git add -A` without checking .gitignore. Credentials (.env, *.pem, id_rsa) can be accidentally committed.

8. **Unbounded numeric config** — lib/config.sh does not validate MAX_REVIEW_CYCLES, CODER_MAX_TURNS, etc. Setting MAX_REVIEW_CYCLES=10000 can cause infinite review loops and massive API costs.
