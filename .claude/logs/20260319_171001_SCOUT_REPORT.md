## Suspected Root Cause Areas

1. **architect.sh (style)** — Two while-loop bodies declare `local cleaned=...` instead of declaring once before loop. Pattern found at lines 218-235 (OOS section) and 280-297 (Design Doc section).

2. **coder.sh (metrics)** — Context component logging happens on line 329 before the notes-gating clear on lines 341-346. When `should_claim_notes` returns false, `HUMAN_NOTES_BLOCK` is cleared to empty string but context report already captured the pre-clear size.

3. **notes.sh (idiom)** — Line 21 uses `echo "$task_text" | grep -qiE` with redundant `-i` flag and avoidable subshell. Should be `grep -qiE '...' <<< "$task_text"` with character classes removed (already cover case-insensitivity).

4. **config_defaults.sh (documentation)** — Line 127 sets `USAGE_THRESHOLD_PCT=0` as a stub for an incomplete feature. No inline comment explains this is groundwork.

5. **notes.sh (behavioral)** — The interaction between `should_claim_notes` gating and `NOTES_FILTER` config creates a silent breaking change. Previously `--notes-filter TYPE` would force notes claiming; now `should_claim_notes "task"` must return true OR `--with-notes` must be set. No migration documentation.
