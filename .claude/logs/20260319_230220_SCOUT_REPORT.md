## Key Symbols
- pick_next_note(tag_filter) — lib/notes.sh:225-263. Returns first unchecked note from HUMAN_NOTES.md in priority order (Bugs > Features > Polish)
- claim_single_note(note_line) — lib/notes.sh:268-301. Marks exactly one note from [ ] to [~], archives snapshot
- resolve_single_note(note_line, exit_code) — lib/notes.sh:308-343. Resolves [~] to [x] (success) or [ ] (failure)
- extract_note_text(note_line) — lib/notes.sh:349-364. Strips checkbox prefix, returns note content with tags
- count_unchecked_notes(tag_filter) — lib/notes.sh:369-399. Counts remaining [ ] notes, optionally filtered by section
- _section_for_tag(tag) — lib/notes.sh:211-219. Helper mapping BUG→Bugs, FEAT→Features, POLISH→Polish
- _escape_sed_pattern(text) — lib/notes.sh:204-207. Helper escaping regex special chars for safe sed matching
- resolve_human_notes() — lib/notes.sh:89-197. Existing bulk operation used by finalize.sh. Will be supplemented (not replaced) by resolve_single_note in HUMAN_MODE path

## Suspected Root Cause Areas
- Section mapping in pick_next_note and count_unchecked_notes depends on exact HUMAN_NOTES.md section heading format (## Bugs, ## Features, ## Polish). Schema validation should occur in M15.4.2 orchestration or earlier in the workflow
- claim_single_note and resolve_single_note use tmpfile + mv pattern for portability but don't validate that mktemp succeeds or that mv completes. Should verify successful write before returning success
- resolve_single_note reconstructs [~] version from original [ ] line internally, assuming the note_line passed is the original. If caller passes [~] or [x] line instead, matching fails silently. Documented but could be fragile if assumptions drift
