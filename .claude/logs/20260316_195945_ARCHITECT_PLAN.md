## Simplification

### 1. Reduce `lib/agent.sh` and `lib/agent_monitor.sh` below the 300-line limit

**Files:** `lib/agent.sh` (343 lines), `lib/agent_monitor.sh` (318 lines)

Both files exceed the 300-line Code Quality blocker enforced by the reviewer
checklist. The prior drift observation requested a split of the original 678-line
`lib/agent.sh` into `lib/agent.sh` + `lib/agent_monitor.sh` — that split was
done correctly. However, both resulting files still exceed the limit:

- `lib/agent.sh`: 343 lines (43 over)
- `lib/agent_monitor.sh`: 318 lines (18 over)

The excess in both files is in documentation headers and multi-line inline
comments, not in logic density. No further structural split is required; the
logic boundaries are already correctly drawn.

**Directed actions:**

`lib/agent.sh` — the tool-profile block (lines 20–60) has one multi-line
explanatory comment per profile. Condense each to a single line. The
`NULL RUN DETECTION HELPERS` banner (lines 290–293) and function-level
`check_agent_output()` usage comment (lines 301–305) can be trimmed to
essential content. Target: ≤ 298 lines.

`lib/agent_monitor.sh` — the file-change detection helpers
(`_detect_file_changes`, `_count_changed_files_since`) carry verbose
parameter/return comments. Condense to match the style of `lib/common.sh`
(single-line descriptions). Target: ≤ 298 lines.

**Constraint:** Do not move functions between files. Do not change function
signatures or logic. Comment reduction only. `shellcheck` must continue to pass
on both files.
