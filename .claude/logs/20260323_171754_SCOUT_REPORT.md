- `init_dashboard()` in lib/dashboard.sh (line 38-55) — Currently creates empty data files but does not copy static HTML/CSS/JS from templates/watchtower/
- Missing templates/watchtower/ directory — Static files (index.html, style.css, app.js) do not exist yet
- pipeline.conf.example — Missing DASHBOARD_ENABLED, DASHBOARD_REFRESH_INTERVAL, DASHBOARD_HISTORY_DEPTH, DASHBOARD_DIR config keys
- HTML/JS data loading — index.html must load data files via `<script src="data/run_state.js">` etc.; must work with file:// protocol
- Tab persistence — app.js must implement localStorage-based tab state persistence across page reloads
- Auto-refresh — app.js must implement conditional refresh (only when pipeline is running)
- Causal trace interaction — app.js must implement CSS-class-based highlighting of event chains (no graph library)
- Responsive design — CSS must support 3 breakpoints (>=1200px, >=768px, <768px) with layout degradation
