## Suspected Root Cause Areas

1. **Architectural cross-concern**: lib/notes.sh shouldn't know about PROJECT_DIR/CODER_SUMMARY.md. This knowledge belongs in stages/cleanup.sh which already understands the project layout.

2. **Portability pattern**: The `wc -l | tr -d '[:space:]'` idiom appears in 33 files. This should either be a shell function in lib/common.sh or documented as a deliberate pattern. GNU wc and BSD wc differ slightly on output formatting.

3. **Context report fragility**: lib/context.sh line 81 and 106 use literal `\n` with `echo -e`. If refactored to use `printf`, this will break. Should use `printf` or `$'...'` syntax during accumulation.

4. **Config defaults inconsistency**: CLAUDE_REVIEWER_MODEL has no default. If pipeline.conf is missing this key and stages/review.sh reads it directly, runtime will fail silently with an empty model string.

5. **Cleanup parsing design choice**: _parse_cleanup_report() intentionally ignores "Not Attempted" items (leaves them `[ ]`), preventing them from being reselected. This is correct but needs a comment to prevent future confusion about the missing case.

6. **Line count context**: LAST_CONTEXT_TOKENS in the final run summary reflects only the most recently completed stage (usually tester), not the coder's context (typically largest). This is a design choice documented only in NON_BLOCKING_LOG.md, not in the code.
