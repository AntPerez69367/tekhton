## Suspected Root Cause Areas

- Non-blocking item selection logic in `select_cleanup_batch()`: must correctly parse NON_BLOCKING_LOG.md format (date + task + file description per line), prioritize by recurrence (grep -c patterns), overlap with modified files (from `git status --porcelain`), and FIFO age. Format string `- [ ] [YYYY-MM-DD | "task"] file description` must be handled carefully.
- Trigger condition in tekhton.sh: must check CLEANUP_ENABLED, count open items via `count_open_nonblocking_notes()`, and verify no stage had null run (check `SKIP_FINAL_CHECKS` is not set).
- Item resolution marking: `mark_note_resolved()` and `mark_note_deferred()` must use `sed -i` safely with literal `[` `]` brackets (escaping in sed patterns).
- Build gate interaction: cleanup stage must run `run_build_gate()` after jr coder completes. Cleanup stage is low-priority — build gate failure should warn but not fail the overall pipeline (unlike coder/tester stages).
- Jr coder invocation: cleanup.prompt.md must instruct jr coder to address items **individually** so each item is independently fixable. The architecture-awareness rule ("[DEFERRED] for architectural changes") is critical — without it, cleanup may attempt unsafe refactors that break the build.
